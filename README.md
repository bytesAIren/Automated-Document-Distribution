# Automated Document Distribution
### Excel + Google Apps Script Integration
#### Learning one mess at a time™

---

## What this is

A small but fully working automation that takes boring repetitive work — copying names, generating PDFs, and sending dozens of emails — and makes Excel and Gmail do it *for you*.

Built by a professional from a commercial back-office background with **zero formal programming training**, but a lot of patience, curiosity, and an LLM on speed dial.

---

## What it does

This workflow handles the "boring stuff":
- **Data Extraction**: Reads data from an Excel or Google Sheet.
- **Document Personalization**: Generates custom PDFs based on different templates (V1, V2, V3).
- **Automated Distribution**: Dispatches individual emails through the Gmail API.
- **Activity Logging**: Keeps a record of everything sent, because chaos is not a strategy.

---

## Tech stack (a.k.a. "tools I learned to master")

- **Microsoft Excel VBA**: For data parsing and local PDF generation.
- **Google Apps Script**: For interfacing with the Gmail API and secure delivery.
- **LLM Assistance**: Used as a tireless co-pilot for drafting code and explaining technical concepts.

---

## Why I built it

In my commercial back-office role, sending 20+ personalized communications manually felt like a crime against productivity.

I wanted to see if I could automate the whole process **without being a developer**. This project is the result of tackling one technical hurdle at a time, turning "internal screaming" into functional scripts.

---

## How it works (The logic)

1. **Excel side**: Reads customer data, picks the right template, and spits out a PDF for every row.
2. **Google side**: Takes that same data and sends the corresponding email via Gmail.
3. **Human side**: Clicks "Run," watches the magic happen, and feels the synapses click together.
